# PWA Skills Trial — Phase 6: Mission Engine & Adaptive Learning

## 1. Anatomy of a Mission
Every mission inside the Skills Trial is defined by a structured schema:

* **Story:** Fictional, operational background setting the purpose.
* **Context:** Fictional client communications, files, and reference material.
* **Deliverable:** Concrete output requested (message draft, document link, walkthrough).
* **Competencies:** Targeted behavior patterns measured.
* **Visible Acceptance Criteria:** Clear criteria the candidate must address (e.g. check date conflicts, use specific template style).
* **Hidden Evidence Targets:** Behavioral events monitored unprompted (e.g. reporting blocker early, using timer correctly).

---

## 2. Scenario Variations
To prevent content leakage (candidates sharing trial answers), the engine uses versioned variants of core missions.
* **Example (Newsletter Sim):**
  * *Variant A:* Date conflict in flyer vs. email.
  * *Variant B:* Broken registration link in kickoff notes.
  * *Variant C:* Missing attachment mentioned in the email body.
* **Result:** Same competencies are evaluated, but the exact data answers differ, forcing candidates to practice active verification.

---

## 3. Specialization Branches
Following Day 2 Core Missions, candidates choose or are assigned a specialization track corresponding to current staffing demands:

* **Communications / Newsletter:** Newsletter layout, editing, client tone, link verification.
* **Project Coordination:** Note triage, task creation, project dependencies, stakeholder status briefs.
* **Grant / Nonprofit Research:** Directory searches, eligibility checking, source documentation, fact verification.
* **Web / QA (Optional/Later):** UI bug checking, content updates, reproduction steps.
* **Design (Optional/Later):** Deck formatting, assets organization.
* **Bookkeeping (Optional/Later):** Ledgers, receipt matching, reconciliation.
