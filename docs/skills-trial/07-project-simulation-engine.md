# PWA Skills Trial — Phase 7: Project Simulation Engine

## 1. Cohesive Story Arc
Rather than working on disconnected checklist tasks, the candidate operates on a single evolving client project:

* **Day 0: Onboarding & Availability:** Set declared work windows and timezone. Acknowledge terms.
* **Day 1: Orientation:** Welcome from Purii, sandbox tour of the task console, study reliability & AI safety policies.
* **Day 2: Client Simulation Kickoff:** Receive a messy kickoff email from "Grace Community Center" about an announcement. Find a date conflict, draft clarification questions, edit the announcement layout, and provide a task status handoff.
* **Day 3: Specialization Challenge:** Execute the selected track challenge (Comms, Coordination, or Research) using synthetic directory databases.
* **Day 5: Thursday Newsletter SOP:** Document a repetitive process (based on a mockup walkthrough) into a numbered SOP, identify bottlenecks, and suggest an operational improvement.
* **Day 5: Live Standup:** Arrive for a 15-minute simulated standup. Provide a concise update (Done, Next, Blocked) or coordinate a reschedule.
* **Day 6: Final Walkthrough:** Record a reflection note explaining decisions, lessons, and how PWA works.

## 3. Dynamic Events
The simulation engine injects realistic interruptions:
* **The Blocker Event:** Fictional client forgets an attachment. The candidate must update task status to "Blocked" and submit a handoff note.
* **The Feedback / Revision Event:** Purii issues a correction regarding the date conflict. Candidate must write a revision plan and resubmit.
* **The Reschedule Opportunity:** Candidate is offered a standup time, with a one-time button path to reschedule responsibly.
